# 청년문화예술패스 추천 챗봇 MVP 공유 안내

## 실행 파일

- `CulturePass_Chatbot_MVP.html`
- `culture_db.js`

별도 설치 없이 `CulturePass_Chatbot_MVP.html`을 더블클릭해서 브라우저로 열 수 있습니다. 단, 실제 크롤링 DB를 사용하려면 `culture_db.js`가 HTML 파일과 같은 폴더에 있어야 합니다.

## 반영한 공유 자료

- `prd_CulturePass.md`의 주요 요구사항 중 온보딩, 예산 표시, 챗봇 추천, 장바구니, 조건 기반 추천 흐름을 반영했습니다.
- `CulturePass_UI.html`의 UI 방향을 참고해 상단 예산 표시, 보라색/청록색 톤, 온보딩 모달, 채팅 영역, 장바구니 패널 구조를 적용했습니다.

## 현재 MVP 범위

- 크롤링 CSV 변환 DB 기반 서울 공연 추천
- 자연어 입력 조건 해석
- 지역, 장르, 분위기, 예산, 관람 시기, 배우명 테스트 필터
- 추천 순위 3위 및 작품별 추천 이유 제공
- 추천 카드에서 장바구니 담기
- 좌석/가격 확정 후 사용 금액/잔여 예산 표시
- 장바구니 작품별 관람 희망일 입력
- 연간 스케줄러 초안 자동 배치
- Google Calendar 등록 링크 제공
- 티켓팅 날짜 입력 시 3일 전 알림이 포함된 ICS 파일 저장

## 테스트 예시

- `올해 가을, 서울에서 하는 코미디 연극 추천해줘`
- `5만원 이하로 볼 수 있는 전시 추천해줘`
- `서울에서 하는 오페라를 보고 싶어`
- `공포나 호러 분위기의 작품 추천해줘`
- `전국 단위로 가을에 볼 수 있는 작품 나열해줘`

## 현재 한계

- 현재 DB는 NOL 서울 공연 CSV를 기준으로 하며, 서울 공연 416개 중 836개에 좌석별 가격표를 `priceTiers`로 반영했습니다.
- 일부 공연은 가격 미노출 또는 수집 실패 상태라 예매처 확인이 필요합니다.
- 평점, 출연진은 아직 추천 로직 검증용 필드입니다.
- 실제 예매/좌석/잔여석/공연 시간은 반영하지 않습니다.
- Google Calendar는 MVP 단계에서 직접 API 연동이 아니라 등록 링크와 ICS 파일 저장 방식으로 대체했습니다.

## DB 갱신 방법

- 크롤링 CSV를 다시 받으면 `prepare_culture_db.py`를 실행해 `culture_db.js`를 재생성합니다.
- 새 NOL 서울 CSV를 다시 받으면 `prepare_nol_seoul_db.py`를 실행해 장르 라벨과 좌석별 가격표가 포함된 `culture_db.js`를 재생성합니다.
- CSV 스키마는 `id, source_key, source_site, category, title, start_date, end_date, venue, benefit, booking_url, image_url...` 순서를 기준으로 합니다.
- 좌석별 가격을 추가할 경우 17번째 컬럼 이후에 `R석 100000원 / S석 70000원`처럼 좌석명과 가격을 포함하면 변환 스크립트가 `priceTiers`로 파싱합니다.
